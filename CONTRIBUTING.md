# Contributing

感谢你为本仓库做贡献。

## 提交原则

- 保持内容面向 Go 后端工程实践
- 优先补充可执行、可检查、可落地的规则
- 避免空泛原则，尽量给出反例、正例和检查方式
- 与现有文档风格保持一致，使用明确的规范词：**必须**、**禁止**、**应该**、**不应该**、**可以**

## 修改范围

- 新增规范时，优先判断是否应补充到现有章节，而不是创建重复主题
- 涉及中文写作问题时，同时检查 [10A-chinese-writing-style.md](./10A-chinese-writing-style.md)
- 涉及质量门禁、检查项或流程时，注意同步更新 [14-comprehensive-checklist.md](./14-comprehensive-checklist.md)

## 提交流程

1. 在独立分支上完成修改
2. 确认文档标题、编号、交叉引用和 README 索引一致
3. 自查术语、标点、链接和示例代码
4. 使用清晰的 commit message 提交
5. 通过 Pull Request 说明修改内容、修改原因和影响范围

## Pull Request 建议内容

- 本次变更解决了什么问题
- 为什么需要这项调整
- 影响哪些章节或团队实践
- 是否需要同步更新相关章节
