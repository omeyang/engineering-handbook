## 关键词约定

本手册中的关键词"**必须**"、"**禁止**"、"**应该**"、"**不应该**"和"**可以**"按照 [RFC 2119](https://www.ietf.org/rfc/rfc2119.txt) 中的描述进行解释。

## 适用范围

本手册适用于所有 Go 后端项目的开发、重构和 Code Review，是团队统一的工程质量标准。
